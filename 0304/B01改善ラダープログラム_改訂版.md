## 1. 空パレット検出機能（LAD_200）【改訂】

### 1.1 変更後変数定義

| 変数名 | アドレス（変更後） | 内容 |
|--------|-------------------|------|
| D700 | D700 | 現在読み取ったRFIDデータ |
| D701 | D701 | 空パレット識別コード（定数） |
| D702 | D702 | 空パレット検出カウンター |
| D703 | D703 | 最終空パレット検出時刻 |
| M1600 | M1600 | 空パレット検出フラグ |
| M1601 | M1601 | PC問合せスキップフラグ |
| M1602 | M1602 | AGV搬出要求フラグ |
| M1603 | M1603 | 通常処理フラグ |
| M1610 | M1610 | 空パレット情報通知（HMIへ） |

### 1.2 ラダープログラム【改訂】

```
┌─────────────────────────────────────────────────────────────────┐
│ LAD_200-1: RFID読取・空パレット判定（改訂）                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [X50: RFID読取完了信号]                                        │
│   └──[MOV D700, D200]→ RFIDデータを作業領域へコピー             │
│                                                                  │
│  [X50: RFID読取完了信号]                                        │
│   └──[CMP D700, D701]──→ M1600: 空パレット検出                 │
│          (読取データ = 空パレットコード?)                        │
│                                                                  │
│  M1600: 空パレット検出                                          │
│   └──[SET M1601: PC問合せスキップ]                              │
│   └──[SET M1602: AGV搬出要求]                                    │
│   └──[RST M1603: 通常処理]                                       │
│   └──[INC D702]→ 空パレット検出カウンター増加                    │
│   └──[MOV D703, 現在時刻]→ 最終検出時刻保存                      │
│   └──[SET M1610: 空パレット情報通知]                             │
│                                                                  │
│  [X50: RFID読取完了信号]                                        │
│  + M1600/: 空パレット未検出                                      │
│   └──[SET M1603: 通常処理]                                       │
│   └──[RST M1601: PC問合せスキップ]                               │
│   └──[RST M1602: AGV搬出要求]                                    │
│   └──[RST M1610: 空パレット情報通知]                             │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────┐
│ LAD_200-2: HMI表示データ設定（改訂）                            │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  M1610: 空パレット情報通知                                       │
│   └──[MOV D710, D700]→ HMI表示RFIDデータ                         │
│   └──[MOV D711, K1]→ HMI表示メッセージコード（空パレット）        │
│   └──[MOV D712, D702]→ HMI表示検出回数                          │
│   └──[MOV D713, D703]→ HMI表示検出時刻                          │
│   └──[SET Y60: HMI表示ランプ点灯]                                │
│                                                                  │
│  [M8000: 運転開始]                                              │
│  + M1610/: 空パレット情報通知OFF                                │
│   └──[MOV D711, K0]→ HMI表示メッセージコードクリア                │
│   └──[RST Y60: HMI表示ランプ消灯]                                │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

## 2. PC通信レスポンス処理（LAD_201）【改訂】

### 2.1 変更後変数定義

| 変数名 | アドレス（変更後） | 内容 |
|--------|-------------------|------|
| D720 | D720 | PC応答コード |
| D721 | D721 | PC応答データ |
| M1630 | M1630 | PC応答受信完了 |
| M1631 | M1631 | PC応答正常 |
| M1632 | M1632 | PC応答空パレット |
| M1633 | M1633 | PC応答エラー |
| M1640 | M1640 | 通常処理継続 |
| M1641 | M1641 | タスク発行可能 |
| M1650 | M1650 | PC通信エラー |

### 2.2 ラダープログラム【改訂】

```
┌─────────────────────────────────────────────────────────────────┐
│ LAD_201-1: PC応答受信処理（改訂）                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [X60: PC応答受信信号]                                          │
│   └──[MOV D720, D220]→ 応答コードを作業領域へコピー             │
│   └──[MOV D721, D221]→ 応答データを作業領域へコピー             │
│   └──[SET M1630: PC応答受信完了]                                 │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────┐
│ LAD_201-2: PC応答コード判定（改訂）                             │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  M1630: PC応答受信完了                                          │
│   └──[CMP D720, K0]──→ M1631: PC応答正常                        │
│          (応答コード = 00?)                                        │
│                                                                  │
│  M1630: PC応答受信完了                                          │
│   └──[CMP D720, K1]──→ M1632: PC応答空パレット                   │
│          (応答コード = 01?)                                        │
│                                                                  │
│  M1630: PC応答受信完了                                          │
│   └──[CMP D720, K99]──→ M1633: PC応答エラー                     │
│          (応答コード = 99?)                                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

```
┌─────────────────────────────────────────────────────────────────┐
│ LAD_201-3: PC応答空パレット時処理（改訂）                       │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  M1632: PC応答空パレット                                         │
│   └──[SET M1601: PC問合せスキップ]                              │
│   └──[SET M1602: AGV搬出要求]                                    │
│   └──[RST M1630: PC応答受信完了]                                 │
│   └──[RST M1631: PC応答正常]                                     │
│   └──[INC D702]→ 空パレット検出カウンター増加                    │
│   └──[SET M1610: 空パレット情報通知]                             │
│   （※LAD_200の搬出処理へ）                                       │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```
