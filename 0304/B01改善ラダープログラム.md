## 1. 空パレット検出機能（LAD_200）

### 1.1 概要

RFID読取時に空パレットを判別し、PC問い合わせをスキップしてAGV搬出処理を実行します。

### 1.2 空パレット定義

| 項目 | 内容 |
|------|------|
| 空パレットRFIDコード | 'E0000000'（固定コード）または任意識別コード |
| 判別タイミング | RFID読取直後 |
| 判別方法 | RFIDデータと空パレットコードの比較 |

### 1.3 処理フロー

```
┌─────────────────────────────────────────────────────────────────┐
│                     処理フロー図                                │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [AGV到着検知]                                                   │
│        ↓                                                        │
│  [RFID読取]                                                      │
│        ↓                                                        │
│  [空パレット判定]────→[空パレット]──→[PC問合せスキップ]         │
│        ↓                                    ↓                   │
│  [通常パレット]                        [AGV搬出指示]            │
│        ↓                                    ↓                   │
│  [PC問合せ]                          [空パレットログ記録]       │
│        ↓                                    ↓                   │
│  [タスク発行]                          [完了通知]               │
│        ↓                                                        │
│  [完了通知]                                                     │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

### 1.4 変数定義

| 変数名 | アドレス | 内容 |
|--------|----------|------|
| D100 | D100 | 現在読み取ったRFIDデータ |
| D101 | D101 | 空パレット識別コード（定数） |
| D102 | D102 | 空パレット検出カウンター |
| D103 | D103 | 最終空パレット検出時刻 |
| M600 | M600 | 空パレット検出フラグ |
| M601 | M601 | PC問合せスキップフラグ |
| M602 | M602 | AGV搬出要求フラグ |
| M603 | M603 | 通常処理フラグ |
| M610 | M610 | 空パレット情報通知（HMIへ） |

### 1.5 ラダープログラム

#### 1.5.1 空パレット識別コード設定

```

│ LAD_200-1: 空パレット識別コード設定（初期化）                   │
├─────────────────────────────────────────────────────────────────┤
│                                                                  │
│  [M8000: 運転開始]                                              │
│   └──[MOV D101, K'E0000000']→ 空パレット識別コード設定          │

## 2. PC通信レスポンス処理（LAD_201）

### 2.1 概要

入出庫PCソフト改修に対応した通信レスポンス処理を追加します。

### 2.2 応答コード定義

| コード | 意味 | B01動作 |
|------|------|---------|
| 00 | 正常（該当データあり） | 通常処理継続 |
| 01 | 空パレット | AGV搬出処理へ |
| 99 | その他エラー | アラーム発生 |

### 2.3 変数定義

| 変数名 | アドレス | 内容 |
|--------|----------|------|
| D120 | D120 | PC応答コード |
| D121 | D121 | PC応答データ |
| M630 | M630 | PC応答受信完了 |
| M631 | M631 | PC応答正常 |
| M632 | M632 | PC応答空パレット |
| M633 | M633 | PC応答エラー |

### 2.4 ラダープログラム

#### 2.4.1 PC応答受信処理

```
┌─────────────────────────────────────────────────────────────────┐
│ LAD_201-1: PC応答受信処理                                      │

│   └──[MOV D121, D221]→ 応答データを作業領域へコピー             │
│   └──[SET M630: PC応答受信完了]                                 │
│                                                                  │

## 6. PCソフト側変更仕様

### 6.1 変更内容

| 項目 | 内容 |
|------|------|
| CSV検索処理 | 該当データなし時に応答コード01を返す |
| 応答処理 | 応答コード01（空パレット）の追加 |
| ログ処理 | 空パレット検出時のログ記録追加 |

### 6.2 通信メッセージ仕様

```
【PLC → PC】
要求: RFID読取データ

【PC → PLC】
応答コード: 00（正常）、01（空パレット）、99（エラー）
応答データ: 正常時はタスクデータ、空パレット時はなし
```

## 7. 実装上の注意点

### 7.1 既存プログラムとの統合

| 注意点 | 内容 |
|--------|------|
| RFID読取タイミング | 既存処理との競合を回避 |
| AGV搬出処理 | 既存の搬出処理と統合 |
| HMI表示 | 既存表示処理との整合性 |

### 7.2 空パレット識別コード確認

| 確認事項 | 内容 |
|----------|------|
| RFID仕様 | 空パレット識別コードの定義有無 |
| コード形式 | 文字列か数値か |
| 複数コード | 空パレット種類によるコード違い |

### 7.3 テスト項目

| テスト項目 | 確認内容 |
|-----------|----------|
| 空パレット検出 | RFID読取〜空パレット判別 |
| PC通信スキップ | PC問合せスキップ動作 |
| AGV搬出 | 空パレット搬出動作 |
| 通常パレット | 通常処理への影響なし |
| HMI表示 | 表示内容の確認 |
