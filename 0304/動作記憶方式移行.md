## 1. 動作記憶方式の概要

### 1.1 方式比較

| 項目 | Step方式（既存） | 動作記憶方式（改善） |
|------|-----------------|---------------------|
| 管理方法 | Step番号による固定シーケンス | 完了フラグによる柔軟管理 |
| 中断時対応 | Step番号修正が必要 | 状態保存・復元が容易 |
| 中間工程再開 | 困難 | 可能 |
| デッドロック対応 | 不可 | 可能 |

### 1.2 基本概念

```
【動作記憶方式の考え方】

工程1完了 → F_1（工程1完了フラグ）セット
    ↓
工程2開始条件: F_1がON
工程2完了 → F_2（工程2完了フラグ）セット
    ↓
工程3開始条件: F_2がON
工程3完了 → F_3（工程3完了フラグ）セット
    ↓
完了処理: F_1, F_2, F_3を全てリセット

中断時: 各フラグ状態を保存 → 復元時に再開
```

## 10. 実装上の注意点

### 10.1 アドレス競合回避

| 注意点 | 内容 |
|--------|------|
| 既存フラグとの整合性 | 既存プログラムで使用中のアドレスを確認 |

| データレジスタ割り当て | 不揮発性領域への保存が必要 |

### 10.2 移行手順

```
【推奨移行手順】

   └─ 現在のStep番号管理箇所を特定

   └─ 新規フラグをアドレスマップに追加

3. シーケンス書き換え
   └─ Step判定をフラグ判定に変更

4. 状態保存・復元追加
   └─ 保存・復元処理を追加

5. テスト
   └─ 各シーケンスの動作確認
```
