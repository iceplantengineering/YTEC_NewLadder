## 6. 工数見積の前提条件

本工数見積は以下の前提条件に基づいています。

| 項目 | 前提条件 |
|------|----------|
| **作業時間** | 1日8時間、月間21日作業 |
| **スキルレベル** | PLCラダー設計・実装の経験者（3年以上） |
| **作業環境** | 開発機・テスト機が使用可能 |
| **仕様確定度** | 現行仕様書の内容に基づき、追加仕様確定後に着手 |
| **レビュー工数** | 設計・実装レビューを含む |
| **試験準備** | テストケース作成・テストデータ準備を含む |
| **文書作成** | 設計書・テスト結果報告書作成を含む |
| **障害対応** | 軽微な不具合修正を含む（大幅な設計変更は除く） |

## 7. リスクと備考

### 7.1 工数変動要因

以下の要因により、工数が増減する可能性があります。

| 要因 | 影響 | 備考 |
|------|------|------|
| **仕様変更** | +10～30% | 実装中の仕様変更 |
| **設備制約** | +20～50% | 現場での試験時間制限 |
| **習熟度** | -10～+20% | 作業者のスキルによる |
| **並列化度** | -30～-50% | 複数名での並列作業 |
| **テスト環境** | +10～30% | テスト環境の不備 |

### 7.2 推奨事項

| 項目 | 推奨事項 |
|------|----------|
| **優先順位** | 最優先（安全対策）→ 高優先（デッドロック防止）の順で実施 |
| **並列実施** | フェーズ1完了後、フェーズ2は複数名で並列実施を推奨 |
| **テスト重視** | 安全に関わる対策のため、十分なテスト時間を確保 |
| **現場確認** | 実機での動作確認を必須とする |

## 8. 結論

* **総工数**: 40.7人日（1.94人月）
* **所要期間**: 約8週間（1名作業時）
* **内訳**:
  - 仕様書明記内容: 15.9人日（約3週間）
  - 追加推奨改善: 24.8人日（約5週間）

* **推奨アプローチ**:
  1. まずフェーズ1を実施し、仕様書記載の改善を実装
  2. 次にフェーズ2の最優先（安全対策）を実施
  3. その後、高優先・中優先の対策を実施

* **注意点**:
  - 安全に関わる変更であるため、十分なテストと現場確認が必須
  - 可能な限り複数名で並列作業を行い、期間を短縮推奨
